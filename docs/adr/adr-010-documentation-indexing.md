# ADR-010: Documentation Indexing

*Status*: Implemented · *Date*: 2026-04-10 · *Context*: As the repo grows, finding the right document becomes harder for both humans and AI. A central index is needed. This approach is dogfooded from the personal repo's indexing strategy.

## Decision

Every markdown file gets frontmatter with a summary and tags. A central `INDEX.md` is generated from this frontmatter.

### Frontmatter format

```markdown
---
summary: This ADR (Application Design Requirements) document outlines the strategy for indexing and searching within the documentation repository of an open-source project. The main goal is to improve the searchability and accessibility of the documentation, especially as the repository grows.  Key points fr
tags: [tag1, tag2, tag3]
---

```text

| Field | Required | Description |
|---|---|---|
| `summary` | Yes | 1 sentence, 200-300 chars, specific and technical |
| `tags` | Yes | 3-5 searchable keywords, lowercase |

### Staleness detection

`make summarize` detects stale summaries by checking `git diff main...HEAD`.
Modified files with an existing summary are re-verified by the LLM: if the
summary still matches the content, it responds "up to date" and no change is
made. This avoids unnecessary commits while keeping summaries accurate.

### INDEX.md

A generated file in the repo root listing all documents with their path and summary. Serves as a table of contents for both humans and AI.

### Generation

- Frontmatter is added manually (or via Ollama for bulk)
- `INDEX.md` is generated by a script that reads frontmatter from all `.md` files
- Regenerated before each release

## Rationale

- AI tools can read `INDEX.md` to understand the entire repo without scanning every file
- Humans get a searchable table of contents
- Frontmatter is a standard pattern (used by Jekyll, Hugo, Obsidian)
- Dogfooding: this approach is used and proven in the personal knowledge repo

## Consequences

- All documentation files should have frontmatter
- A `scripts/dev/build-index.sh` script is needed (future)
- `INDEX.md` should be in `.gitignore` if generated, or committed if curated
